SEO的形像比喻

2011年03月17日

  搜索引擎如何决定众多站点的权重及页面的排名？搞计算机的都知道，搜索引擎本质上是一个程序，是开发人员将算法规则用程序来进行模拟。我们可以把的网站比做社会中一个一个的人，搜索引擎要做的，就是要识别这些人，并按照要求，快速地找到你需要的人。

  我们知道，很多站点下都有友情链接，这个将被视为站外导出，如果一个站被其它站在友情链接中引用较多，权重就会高一些。就好比一个人被其它人提及得越多，就说明这个人社会关系网越广泛，社交能力强，受到大家的欢迎。

  当然，如果你被一些“垃圾”站点大量引用，可能会降低你的权重。就好比一个人的朋友都是些臭名昭著的人，也就说明这个人不是什么好人，中国有句话叫“近朱者赤，近墨者黑”。反之，如果你被一些大人物、名人时常提起，那么你的地位也一定很高。

  站点的外链也和上面讲的同理，如果有很多站都引用你的站点或站内页面地址，也将证明这个站点或站内页面权重高。你可以看看开心网，虽然只有一个简单的首页，但权重很高，还有一个例子，工信部的网站是中国惟一一个google pr达到9的站点，原因也在于众多的网站都有这个站的外链。当然，搜索引擎也会判断是哪些类型的网站引用了你，就好比你是搞IT的，但是被一些搞艺术的人经常提起，这就说明有问题，网站排名会受到影响。“物以类聚，人以群分”就是这个道理。

  说完了一个人的社会关系，再来看看一个人自身的人品。首先是一个网站的结构要清晰，重点突出，让人一目了然，这就好比一个人，要把自己推销出去，自身的素质要提高 ，要有良好的表达，着装工整，要有亲和力。站内的内部链接要做好，这个是结构的问题，但可以提高与搜索引擎的亲和力。站点的美化要做好，也就是美工、用户体验，现在搜索引擎越来越重视这个了，google即然有自己的浏览器，也一定可以评测出站点在浏览器中的显示效果，最近baidu也出浏览器了，搜索引擎从技术上已经可以给页面的美观评级了。

  网站内容原创最好，不要从其它网站抄过来再改改（伪原创），也就是一个人，要有自己的性格和魅力，要有自己的特点、气质，如果硬是从其它地方搬过来一些东西再包装一下，但这些东西终究不属于自己，骗得了一时骗不了一世。抄袭，也证明了这个人的人品有问题，如果人品出现问题，这个人将很难在社会中立足。

  站点要经常更新（更新频率），如果一个人“顽固不化，一成不变”，自然得不到其它人的关注。“喜新厌旧”的道理大家都应该明白。

  搜索引擎就好像根据这些规则去相识、相知一个人，直至建立起信任关系，只不过这一切是程序自动做的，当你要找某人时，搜索引擎将尽可能找到最符合条件的人给你，这也就是搜索引擎公平、公正的原则。

服务器回来了

2011年03月09日

  电信又开始严查了，先前托管的一台服务器被“赶”出了机房。一年后又见到了服务器，最近要开始准备数据迁移了。

  由于托管的服务器机房较远，在平日的操作中都极为谨慎，生怕将系统搞瘫惹出多的事情，感觉就像发射出去的一颗卫星，要是出现故障维护可就难了，呵呵。不过，现在这颗“卫星”已经收回来了。

  打开箱子取出服务器，居然发现服务器非常的干净，透过机箱缝隙可以看到内部电路元件上几乎无灰尘，就像新购置的一样，不禁感叹电信机房的环境（看来真的是无尘），你要想想，服务器是24小时不间断工作了一年多哈，发张照片上来，呵呵。

我的服务器

  数据转移时遇到当初系统一个设计上的问题，当初上传图片、音频、视频等资源的存放目录没有做规划，导致现在将资源重新上传到新的服务器上时，无法挑出几年前的老资料做特殊处理，这点，请大家一定要注意，在进行系统设计时，资源一定要归好类，一般按资源类、年、月建目录存放就非常不错了，比如：/var/data/projectname/video/2011/03/，用来存放2011年3月份的视频资源。

Java 图像缩放方案

2011年02月18日

  java对图片的操作功能可谓是比较弱，而且效果比较差。而在项目中，又经常会对图片进行操作，比如：用户上传一个图片上来做为头像，系统就要对图片进行缩放，以使头像符合界面显示要求，并且减少头像文件大小，对站点的响应有一定作用。

  图片绽放，精度肯定会损失，一般我们都要求缩放后的图片在视觉上平滑，不要出现锯齿，下面列举一些实现方案：

  一、利用j2se的Graphics、JPEGImageEncoder等类实现图片缩放，这样的代码网上很多，有兴趣的可以google一下。这种方案的缺点是：效果不理想。还有一点，JPEGImageEncoder在有些jdk实现版本中不支持，比如：OpenJDK。

  二、借助于开源图像处理框架Java Image Filters，网址在这：http://www.jhlabs.com/ip/filters/index.html，该框架可以实现常见的图片处理，在http://www.oschina.net/bbs/thread/8248还有一篇文章，介绍如何使用。不过，你打开源代码看看图片缩放的源码，就会发现框架也是简单的封装了j2se的图像处理api，因些，缩放效果很不理想。当然，该框架中的其它图像处理功能比较强大。

  三、使用 ImageMagick,这也是一个开源框架，c语言写的，还有专门的java的接口，在http://www.imagemagick.org /script/api.php可以看到ImageMagick的各种语言接口实现。该框架的功能非常强大，图像处理后出来的效果是最好的，像是缺点是：环境搭建困难，比如：在windows下需要dll,jar包的支持，如果是linux或unix系统要手动编译.so的库文件，新手搞起来肯定比较困难。
