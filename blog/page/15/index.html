艰难的互联网创业

2011年03月22日

  中国的互联网已经进行几大巨头独霸的局面，中小创业者在夹缝中艰难地生存着，创新很容易被模仿，即时有创新，凭自己的力量实现并推广也是比较困难的。总体来说，中国互联网缺少法律保护，抄袭、同质化问题比较严重。下面说说我身边发生的一些事情，有些是几年前的。从这些事情中，你也许可以读到想要做好一件事，哪些因素比较关键。

  先说老彭，老彭的思路非常灵活，而且很多东西有自己的想法，但由于种种原因，这些创意都没能实现。他先是提出要做一个站点实时监控网站，专门为中国互联网的广大站长提供站点实时监测，比如：每日浏览，故障次数，平均响应时间等，这些信息会通过手机短信的方式发送给付费站长，普通用户仅有邮件通知。这个想法提出来时，最大的问题如何采集到不同操作系统服务器上的数据，以及如何开发更深层次的扩展应用，比如：站点流量，新增注册用户数等插件开发。想法是好，经过网上查询，当时admin5已经有一个类似的项目，也应该有不错的用户数量，再做一个同类的项目，可以说无法和admin5竞争，我们都知道，“先入为主”这句话在互联网来说非常合适，别人即使比你先推出来一天，效果都是非常不一样的，这个想法就这样被否定。现在，“监控宝”是这类站点中的佼佼者，admin5的监控已经落败，监控宝是有的强大的后台财力支撑，而且做了很多创新，以当时老彭的实力，实现也比较困难。后来，老彭提出做一个saas类的项目，专门供美容美发店使用（会员卡、每日运营情况等），按店收取年费，我都见他设计完了整个项目，包括领域，er，详细设计，后面不知什么原因放弃了，也许问题在推广上。再后来，想做艺术品交易网站，连盈利模式这些都已经想得比较清楚了，但后来也放弃了，原因在于网上有同质化的网站，你想嘛，如果淘宝也开一个这样的版块，那我们就没得搞头了。再后来，想做一个星巴克的网上门店，这个想法因盈利模式有问题放弃。再后来，计划做一个gay交流网站，兼gay物品交易，这个想法应该是做过前期调查的，不知什么原因没了消息。再后来，想做客运站的一个物流系统，也没听到有消息。

  老杨，先是做团购网站，但线下营销能力不足，效果不好，最后网站因备案问题进入电信机房黑白名单过滤而关闭。后来想做外包网站，我说已经有这样的网站了，但他说在很多地方可以改进及创新，当时考虑到开发效率，用php实现，最后也没听到他的这个网站的消息了。看来做技术的人大多都是重技术而不精营销的。前段时间，我还听说某些软件公司的技术人员想用业余时间做团购网站，也许他们真的不知道什么是线下营销，我们要知道，挣钱的网站，大多是用站点做门户辅助配合线下营销，一个团购网，技术上应该是很容易实现的。

  我们老大，有很多不错的创意。首先是小孩防疫情况通知的这个系统，全国有小孩的家庭应该还是很多的，这个项目要是做成并成功推广，应该是非常不错的。网络智能发贴跟贴（自动注册，发贴，跟贴，而且要有一定的人工智能性），这个做好了也会有收不错收入。电子书平台，进行在线学习，可以将各种格式文档转换为网站内部统一格式，实现在线教育培训，这个项目好像进行了一段时间就终止了，有点可惜。

  肖老二写了一个“烂笔头”，收取注册费，也似乎在互联网上推广过一段时间，最后没了消息。其实，这类软件一定要有功能亮点，否则无法盈利。还有一点，中国网民已经习惯了免费，不想掏钱（玩游戏除外哈），这点根深蒂固。

  军哥，技术牛人，狂人，本人很佩服他，也有一些不错的想法，最后都未能被实现，我们也知道，人家的主业可是专业炒股，专业做投资的哦。我身边有投资意识的人，目前仅此一人，我遇到一些问题，都还得向他咨询。

我的一个做互联网的朋友，也比较有想法。一年前申请了一个域名（墓地网），准备做网上墓地销售，不知什么原因，该域名如今还空着的。后来还做过一个比较有前途的“二手工程机械交易网”，但由于线下营销能力跟不上来，最后没了消息。

  我的另一个朋友，非常精于运营和营销，自己带团队做过网页游戏，最终失败于技术。此人的运营营销能力非常强，就像天生是为这行而生的，我从那里学到了很多的东西，也深刻地意识到，技术如何快速适应需求，为需求服务，如果按“软件工程”思想按部就班地做互联网项目，只有死路一条。

  最近一同事向我提起曾今想做一个旅游网站，专为外国人提供散客或组团导游服务，我觉得这里面很多事情没有考虑清楚，盈利方式不明确。我还有一个新疆的同学，上回QQ上给我说想做一个闭路电视在线交费系统，要和广电上做接口什么的，说了很多，感觉信心很大，但我认为这个项目对现在的广电来说基本上没有什么利益和好处，也行无法推广，这个想法也是“虎头蛇尾”，没了下文。

  我也有很多想法，但实现的不多，我对互联网本来就缺乏敏锐性，但我认为互联网上创业的机会还是有的，在互联网上做事情，可能比现实社会中做事要单纯得多，更适合技术人员一些，但是要做强做大，技术人员就不适合了。我认为要将自己的创意变为现实，要注意下面几点。

  1、想好盈利模式；

  2、务实，脚踏实地，避免不必要的投入（比如计算后期用户量有多大，流量有多大，等等，），避免好高骛远；

  3、有想法，要有实现的动力，要用恒心；

  4、如果有团队，一定要统一每个人的预期目标，这点很重要；

  5、要有合作精神，靠一个人的力量很难做成大事；

  6、坚持，坚持，再坚持。在目标没有达到之前，请不要轻言放弃。

SEO的形像比喻

2011年03月17日

  搜索引擎如何决定众多站点的权重及页面的排名？搞计算机的都知道，搜索引擎本质上是一个程序，是开发人员将算法规则用程序来进行模拟。我们可以把的网站比做社会中一个一个的人，搜索引擎要做的，就是要识别这些人，并按照要求，快速地找到你需要的人。

  我们知道，很多站点下都有友情链接，这个将被视为站外导出，如果一个站被其它站在友情链接中引用较多，权重就会高一些。就好比一个人被其它人提及得越多，就说明这个人社会关系网越广泛，社交能力强，受到大家的欢迎。

  当然，如果你被一些“垃圾”站点大量引用，可能会降低你的权重。就好比一个人的朋友都是些臭名昭著的人，也就说明这个人不是什么好人，中国有句话叫“近朱者赤，近墨者黑”。反之，如果你被一些大人物、名人时常提起，那么你的地位也一定很高。

  站点的外链也和上面讲的同理，如果有很多站都引用你的站点或站内页面地址，也将证明这个站点或站内页面权重高。你可以看看开心网，虽然只有一个简单的首页，但权重很高，还有一个例子，工信部的网站是中国惟一一个google pr达到9的站点，原因也在于众多的网站都有这个站的外链。当然，搜索引擎也会判断是哪些类型的网站引用了你，就好比你是搞IT的，但是被一些搞艺术的人经常提起，这就说明有问题，网站排名会受到影响。“物以类聚，人以群分”就是这个道理。

  说完了一个人的社会关系，再来看看一个人自身的人品。首先是一个网站的结构要清晰，重点突出，让人一目了然，这就好比一个人，要把自己推销出去，自身的素质要提高 ，要有良好的表达，着装工整，要有亲和力。站内的内部链接要做好，这个是结构的问题，但可以提高与搜索引擎的亲和力。站点的美化要做好，也就是美工、用户体验，现在搜索引擎越来越重视这个了，google即然有自己的浏览器，也一定可以评测出站点在浏览器中的显示效果，最近baidu也出浏览器了，搜索引擎从技术上已经可以给页面的美观评级了。

  网站内容原创最好，不要从其它网站抄过来再改改（伪原创），也就是一个人，要有自己的性格和魅力，要有自己的特点、气质，如果硬是从其它地方搬过来一些东西再包装一下，但这些东西终究不属于自己，骗得了一时骗不了一世。抄袭，也证明了这个人的人品有问题，如果人品出现问题，这个人将很难在社会中立足。

  站点要经常更新（更新频率），如果一个人“顽固不化，一成不变”，自然得不到其它人的关注。“喜新厌旧”的道理大家都应该明白。

  搜索引擎就好像根据这些规则去相识、相知一个人，直至建立起信任关系，只不过这一切是程序自动做的，当你要找某人时，搜索引擎将尽可能找到最符合条件的人给你，这也就是搜索引擎公平、公正的原则。

Java 图像缩放方案

2011年02月18日

  java对图片的操作功能可谓是比较弱，而且效果比较差。而在项目中，又经常会对图片进行操作，比如：用户上传一个图片上来做为头像，系统就要对图片进行缩放，以使头像符合界面显示要求，并且减少头像文件大小，对站点的响应有一定作用。

  图片绽放，精度肯定会损失，一般我们都要求缩放后的图片在视觉上平滑，不要出现锯齿，下面列举一些实现方案：

  一、利用j2se的Graphics、JPEGImageEncoder等类实现图片缩放，这样的代码网上很多，有兴趣的可以google一下。这种方案的缺点是：效果不理想。还有一点，JPEGImageEncoder在有些jdk实现版本中不支持，比如：OpenJDK。

  二、借助于开源图像处理框架Java Image Filters，网址在这：http://www.jhlabs.com/ip/filters/index.html，该框架可以实现常见的图片处理，在http://www.oschina.net/bbs/thread/8248还有一篇文章，介绍如何使用。不过，你打开源代码看看图片缩放的源码，就会发现框架也是简单的封装了j2se的图像处理api，因些，缩放效果很不理想。当然，该框架中的其它图像处理功能比较强大。

  三、使用 ImageMagick,这也是一个开源框架，c语言写的，还有专门的java的接口，在http://www.imagemagick.org /script/api.php可以看到ImageMagick的各种语言接口实现。该框架的功能非常强大，图像处理后出来的效果是最好的，像是缺点是：环境搭建困难，比如：在windows下需要dll,jar包的支持，如果是linux或unix系统要手动编译.so的库文件，新手搞起来肯定比较困难。

Java Mis类型项目招聘技术点总结

2011年01月20日

  我有过一些在软件公司招聘的经历，下面谈谈在招聘过程中我所关注的一些地方，做为新手可以从这些方面去掂量下下自己，找一个合适的方向。

  当然，不同的公司，招聘条件并不相同，不同的项目，招聘条件也不同。但是也存在一些招聘误区，比如：有的公司喜欢作秀，把题目出得很难很深，这些考试内容在实际项目中根本用不上；也有的公司把自己花了很大精力才解决的问题拿来考面试者；还有的公司出一堆IQ测试题等等，这样的情况非常多。我还是喜欢务实一些的，工作中、项目中用什么就考什么。据我所知，成都的很多公司喜欢来虚的，深圳、上海等地的公司更务实些，呵呵。

  我以前出的一些题目，主要是从实际项目出发，考核应聘者的一些基本能力，不同类型的项目，条件也不一样，但有很多基础的东西是必须要掌握的，下面以开java开发MIS系统为例，进行讲解。

  1、java基础

  a、基本语法：顺序、分支、循环这三种基本的程序结构要了解，同时要关注一下执行效率问题(比如：变量的申明范围、类型等)。

  b、对象的继承、封装、多态，对象修饰符，构造函数等知识。

  c、集合的使用：数组，set，map，list的基本用法。

  d、基本流操作：InputStream,OutputStream及几个常用子类，比如：操作文件的、操作带缓冲区的、带整行读取功能的、操作字节的等，流操作虽然用得不多，但有时还是要用的哦。

  e、多线程原理。对理解服务器请求处理及资源同步有帮助。

  f、工具类的使用：有这么几个类很常用SimpleDateFormat，Calendar，Math，Radom，字符串与数字类型的转换，以及正则达式类。其实也就是不同类型的转换、格式化等操作。

  2、web部分

  a、socket的原理，http头（很重要）。这些东西在实际编码时几乎不用，但又需要程序员理解，因为这些知识对解决实际问题非常有帮助。

  b、4个web作用域，必须要有一定的认识。session与cookie的联系区别要清楚。

  c、页面与业务的交互流程要清楚，还要了解一些安全交互方面的知道，比如：数据验证，跨域等。

  d、el和jstl，jstl中的core，fmt和fn都要熟悉，特别是日期、数字的格式化。

  e、对xml解析方式要清楚，有的项目在这方面要求还比较高。

  3、页面展现

  a、基本html元素必须熟悉。

  b、合理使用css定制页面外观。

  c、基本的js操作方法（取页面元素、加载事件、触发事件等）。

  e、一些js框架，比如：jQuery，Extjs等，其中jQuery是重点哦。

  f、父、子页面（或iframe）间的参数传递及互访。

  g、ajax交互原理。

  3、数据库方面

  a、基本的sql语法，特别是子查询、分组，聚合、分页。

  b、特定平台的存储过程（有的项目需要）。

  c、范式的基本认识。

  d、一些性能方面的知识。

  4、框架部分知识

  a、struts1.x，重点是一个交互流程。DispatchAction，FormBean的基本用法。

  b、hibernate,关系映射的配置是一个方面，另一个是HQL的写法，特别是关联对象，多对多对象的hql写法。hibernate性能方面要有所了解。

  c、对spring容器获取的方法，特别是在web环境下的获取。对spring的AOP、数据库事务管理（重要）以及bean的生命周期要有一定了解。其实spring本身的配置和使用还是相当简单的，重要的是原理。

  e、对其它框架要了解，比如：struts2，ibatis,ejb3,webservice相关框架等等。

  5、架构及设计模式

  架构方面对于一个普通程序员，要求不是太高，但常见的mvc、失血模型、事务脚本、dao这几个，一定要理解透彻

  设计模式只做一般了解，工厂、单例、装饰、适配器、观察者、门面 这几个模式相对来说比较重要，因为模式体现了面向对象设计原则，对代码重构及理解其它写人的代码都有好处。

  上面的这些考核知识点比较散，也不完善、不成体系，但却是项目中所需要的，是从实战出发所总结，大家要引起重视。

  最后，还有两点也是一个公司比较看重要的：

  1、人品，所谓“有德有才 破格重用，有德无才 培养使用，有才无德 限制录用，无德无才是坚决不用”，说的就是德，可见，人的品德是多么的重要。

  2、对技术的敏感性。

  我曾经面试过一个法律专业的小伙来公司做技术，面试时就发现该人非常踏实，诚实，人比较聪明，此人已经自学了一些编程知识，但当时老板不太同意，最终还是招进了公司。这个小伙子我只带了一个月，就很快能够独顶一面，进步得非常快。也许有的人天生就是吃这碗饭的。

  软件行业中对技术还比较迷茫的人，希望读完本文后，能给你点启示，毕竟过来人都走过一些弯路的，这些总结对你一定有用。

  网页的爬取及解析，应该利用一现有的工具或框架去实现，而这些框架的选择，至少要考虑下面的一些因素：

  1、应该具有一定的兼容性，比如：原始页面中的一些语法错误、未关闭标签等，应该能够被正确解析；

  2、应该可以方便地查找、过滤页面元素；

  3、应该要以对找到的元素进行操作，比如：修改，删除等，当然也可以加入新的元素。

  我主要说两个常用的网页解析工具，一个是htmlParser，这个框架出来得比较早，并且有比较广泛的应用，官方站点在http://htmlparser.sourceforge.net/，下面是一段示例代码：

  3、页面的修改、重组都不方便。

  直到有一天，发现了jsoup，以前困扰过我的问题终于得到了解决，并且解决得很彻底，是到了跟HtmlParser说再见的时候了。jsoup官网地址在http://jsoup.org/，整个jar包135kb，非常小巧，而且不依赖其它包，够强悍。下面贴出一段示例代码（具体使用官网上有Cookbook，另外chinaos版主也写了一篇jsoup的使用文档，自己去google一下吧）：
